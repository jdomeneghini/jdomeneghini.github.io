Detecção de fraude em cartão de crédito

A clonagem de cartão de crédito é a fraude mais praticada da atualidade, de acordo com o Serviço de Proteção do Consumidor, gerando impactos financeiros consideráveis tanto para o consumidor quanto para as instituições financeiras.
Diante disso, o investimento em áreas de detecção de fraude por meio de inteligência artificial vem crescendo a cada ano e a segurança nas operadoras de cartão está cada vez mais rigorosa.

Objetivo

O desafio deste projeto será trabalhar em cima de um dataset de grande volume, criando e testando um modelo para que possa prever e evitar possíveis transações fraudulentas.

Dados utilizados

Os dados que usarei neste projeto foram disponibilizados por algumas empresas européias de cartão de crédito. O dataset representa as operações financeiras que aconteceram no período de dois dias, onde foram classificadas 492 fraudes em meio a quase 290 mil transações.

Este é um conjunto de dados extremamente desbalanceados, onde as fraudes representam apenas 0,17% do total. Eles também passaram por uma transformação PCA (Principal Component Analysis), onde os dados foram descaracterizados por razões de confidencialidade.

Qual o impacto dos dados desbalanceados?

Repare no gráfico abaixo:

Nossa classe de fraude (Classe= 1) mal é vista, tornando nítida uma discrepância entres os dados.
Esse tipo de situação é muito comum em casos de análises de fraudes, devido a classe minoritária representar normalmente até 5% do todo. Neste caso, como informado acima, a classe minoritária está representando somente 0,17% do total.

Na prática aplicando estes dados num modelo de classificação, ele apresentará muitos falsos negativos, já que o modelo possuirá mais entradas de transações normais e poucas de transações fraudulentas afetando a capacidade de precisão do mesmo, levando os bancos a terem um grande prejuízo financeiro.

Existem alguns métodos para tratar os dados desbalanceados, basicamente estes métodos equalizam a quantidade de classificações desbalanceadas, mas não entrarei a fundo neste artigo.

Para este projeto para tratar e balancear meus dados, utilizei o método Undersampling. Ele eliminará aleatoriamente a minha classe maioritária (transações normais) afim de equalizar com a minha classe minoritária (transações fraudulentas).

Abaixo podemos visualizar a mudança após aplicar o undersampling:

Com os dados equalizados podemos prosseguir com a criação do modelo.

Modelo - Algoritmo de Regressão Logística

O modelo escolhido para este caso foi o de Regressão Logística, é um algoritmo de machine learning muito utilizado para problemas de classificação, nele os objetos são classificados em dois grupos ou categorias.

Para verificar os resultados obtidos através deste algoritmo e medirmos sua eficácia usamos algumas métricas de desempenho, como a matriz de confusão, onde ela indica os acertos e erros do modelo.

Através dessa matriz de confusão podemos verificar a taxa de previsões do nosso modelo, os resultados em Falsos Positivos foram bons e já garantem uma maior segurança.

Abaixo vamos visualizar nosso relatório de classificação e acompanhar outras métricas de avaliação.

O Recall nos indica que dentre todas situações de classe positivo como valor esperado, quantas estão corretas. Ou seja, o modelo deve de qualquer maneira encontrar todas as transações fraudulentas, neste caso ele atingiu um valor de 93%.

Já a AUC ROC (área sobre a curva) avalia a qualidade das previsões do modelo, um modelo com previsões 100% erradas tem uma AUC de 0, enquanto as previsões 100% corretas tem uma AOC de 1, neste caso ele atingiu o esperado chegando a 0,9514.

Considerações Finais

Apesar dos problemas encontrados neste dataset, com os dados bem tratados e limpos podemos ver que o modelo atingiu um bom resultado e apesar deste resultado, sempre há espaço para mais testes e melhorias afim de alcançar um ótimo resultado.
